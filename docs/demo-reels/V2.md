# Demo Reels V2

V2 demo reels replace the V1 Playwright-driven recording pipeline with a fully
synthetic Remotion render. Same outputs (mp4 + cover JPEG + story copy) and
the same platform post path (`/api/social/post?type=demo-reel`), but the way
the video is built is completely different.

V1 ran a real browser session against prod chunkycrayon.com via Playwright,
recorded the screen, and composited the recording into a final short. V2
renders the entire reel in code as a React composition, no Playwright, no
recording. Three input variants share the same comp skeleton:

- **TEXT** — kid types a short prompt into a textarea
- **IMAGE** — kid uploads a photo, and a coloring page of it appears
- **VOICE** — kid says aloud what they want, two-turn voice conversation

This doc covers V2's architecture, the migration decisions from V1, and the
gotchas that bit us during the build (so future-you doesn't re-discover them).

## Architecture

```
Vercel cron (06:00 UTC, daily)
  -> POST /api/social/demo-reel/produce-v2     (web app, prod)
       1. Auth (Bearer CRON_SECRET)
       2. Pick variant from rotation: read most-recent row's demoReelVariant,
          pick the next in TEXT -> IMAGE -> VOICE -> TEXT cycle.
          (Override available via ?variant=text|image|voice)
       3. Generate the source coloring image:
          - text/voice: generateDemoReelImageFromAIDescription
                        (Perplexity Sonar -> Claude cleanup -> GPT Image)
          - image:      generateDemoReelImageFromPhotoUrl
                        (kid-safe photo from photo_library_entries
                         -> vision analysis -> GPT Image with reference)
       4. POST http://157.90.168.197:3030/publish/v2
          Hetzner worker (chunky-crayon-worker.service)

      Worker /publish/v2 (~5min wall-clock per reel):
       1. Poll DB until row has svgUrl, url, regionMapUrl, regionsJson
          (other generators run async; produce-v2 fires before they finish)
       2. Parse regionsJson (Postgres `text` column, comes back as a string)
       3. Pick the healthiest palette variant (pickBestPalette)
       4. Resolve the kid-typed short prompt via /api/dev/shorten-prompt
       5. Generate per-render audio bundle:
          - ambient music (proxy row.backgroundMusicUrl through localhost)
          - kid + adult voiceover via ElevenLabs (text variant)
          - adult-only reaction line (image variant, see "Image variant audio")
          - Q1 + Q2 + A1 + A2 conversation audio (voice variant)
       6. Branch on variant -> renderTextDemoReelV2 / renderImageDemoReelV2
          / renderVoiceDemoReelV2 (all three share the same Remotion bundle)
       7. Build cover JPEG: gunzip region map, paint each pixel with the
          palette colour, composite SVG line art on top with multiply blend
       8. Upload mp4 + cover JPEG to R2
       9. Write demoReelUrl + demoReelCoverUrl + demoReelStoryUrl
          + demoReelVariant back to the row

Vercel cron (17:00/18:00 UTC weekdays, slightly earlier weekends)
  -> GET /api/social/post?type=demo-reel&platform={instagram|facebook|tiktok|pinterest}
       reads latest row with demoReelUrl, posts to each platform with
       per-variant captions (variant-aware + AI-free)
```

## Migration decisions (V1 -> V2)

### Why drop Playwright recording

V1's Playwright session ran on prod chunkycrayon.com, with the worker
controlling a real Chromium tab. Worked, but:

- Every CSS regression on the live site broke the next render. Production
  brittleness for a marketing pipeline.
- Frame-deterministic rendering was impossible (real-time recording always
  has timing wobble). Couldn't scrub.
- Hard to debug. When a render came out wrong, the only signal was a 7-min
  mp4 and a Playwright trace.

V2 renders the whole reel as a React composition in Remotion. Same fonts,
same spring config, same brand tokens. Frame N is the same image every time,
forever. Studio mode lets us scrub a render before deploying it.

### Why three variants share a bundle

All three reels (`TextDemoReelV2`, `ImageDemoReelV2`, `VoiceDemoReelV2`) live
in the same Remotion bundle (registered in `Root.tsx`). The worker calls
`bundle()` once per render, then `selectComposition(id)` to pick which reel
to actually render. Saves cold-start + bundle-cache time when multiple
variants render in succession (manual backfills, not normal cron path).

### Why the worker polls the row instead of failing fast

`produce-v2` fires the worker request 3 seconds after creating the
coloring image row, but the derived assets (svg trace, region store,
colored reference) are generated by **other worker endpoints** asynchronously
over the next 3-5 minutes. The render needs all of them.

V2 worker polls the row every 5s for up to 8 minutes until svgUrl, url,
regionMapUrl, regionsJson are all populated. The Vercel produce-v2 caller
gives up at 45s but the worker keeps running and writes back when done.

### Why we generate a JPEG cover from the region map (not coloredReferenceUrl)

`coloredReferenceUrl` is the GPT-Image AI-rendered colored version of the
line art. It's stylistically different from what the reel's Magic Brush
sweep reveals (different colour choices, lit scenes, painterly strokes).
Using it as the cover would mismatch the thumbnail with the video viewers
tap into.

V2 cover instead reproduces the runtime canvas composite V1 captured
(drawing canvas + line art with multiply blend) but in pure Node:

1. Fetch + gunzip `regionMapUrl` -> `Uint16Array` (pixel -> region id)
2. Fetch `regionsJson` -> region id -> palette hex lookup
3. Walk every output pixel, paint it the region's palette colour
4. Rasterize the SVG line art at 1024 via sharp at high DPI
5. Multiply-blend SVG over the region fill, JPEG out

Pinterest requires JPEG, IG/FB accept any format. Output is ~280-380KB.

### Why we pick the palette dynamically

The region-store labelling LLM occasionally collapses an entire palette
variant to a single hex value (we've seen 538 regions all assigned the same
"Sunshine Yellow"). Different variants fail on different rows: we've seen
Bumblebee with `cute=1, pastel=1, surprise=1` but `realistic=13`, and
Wildflower with `realistic=1` but `pastel=9`.

`pickBestPalette` defaults to `realistic` (most natural-looking thumbnail
for social) but falls back to the variant with the most distinct hexes if
realistic has fewer than 3. Every render logs the choice + counts so we
can spot when the labelling pass collapsed something.

The cover and the reel reveal use the same palette pick, so the thumbnail
matches what plays in the video.

### Why the kid-typed prompt is generated by Claude (not derived from title)

Live users type natural-short prompts ("a koala on the beach") into the
textarea. The daily AI-generated images have long descriptive titles
("Cute Bumblebee Garden Coloring Page") and even longer scene descriptions.
Typing the long description into the textarea would feel artificial.

`/api/dev/shorten-prompt` runs Claude haiku with the same Bluey-tone
guardrails as the legacy `next-scene-prompt` shortener, returns ~8 words.
Worker calls it once per render, uses the result for both the typed text
AND the kid voiceover. Live shortener lives on the web app to keep one
source of truth and stay close to the Anthropic key.

### Why image variant uses adult narrator instead of kid voiceover

For text + voice variants, the kid voice reads the prompt verbatim because
the kid is the speaker (`kidLine = ${tag} ${prompt}`). For image variant,
the kid uploaded a photo, they didn't describe it, so a kid reading the
description doesn't make sense.

V2 image variant generates an adult-narrator reaction line ("Oh, a
beautiful meadow full of tall flowers — let's turn it into your coloring
page") and plays it during the upload phase, where the kid voice would
normally play. The comp prop is still called `kidVoiceUrl` because the
audio Sequence slot is identical, only the speaker behind the file changes.

### Why image variant voiceover is delayed to frame 60

The narrator was originally starting at frame 0 of Beat 3 (drop-zone empty,
photo not yet uploaded). Reel played as "hear the reaction, then see the
photo" instead of "see the photo, then hear the reaction." Wrapped the
audio in `<Sequence from={60}>` so it triggers when the photo finishes its
drop-in animation (frames 30-60 = preview-progress interpolation).

### Why captions are variant-aware and AI-free

Two issues with V1 captions:

1. Bluey-style brand voice referenced "AI draws the coloring page" and
   `#aiforkids` hashtag. Per memory, parents are AI-skeptical right now;
   AI mentions in marketing copy reduce trust.
2. Single set of captions ignored which input variant the reel actually
   showed. TikTok said "kid types" even on the IMAGE/photo-upload variant.

V2 caption generators read `coloringImage.demoReelVariant` and emit a
hard-required line-1 hook based on the input action (typed / uploaded a
photo / said aloud). Banned words list: AI, artificial intelligence, tech,
automatic, magic brush. Replaced #aiforkids with #screenfreeplay and
similar parent/creativity tags.

## Gotchas

### regionsJson is text, not jsonb

The Prisma schema declares `regionsJson String` (text), not `Json`. Prisma
returns it as a raw JSON string, not a parsed object. Always
`JSON.parse(row.regionsJson)` before passing to anything that expects
`{ regions: [...] }`. We hit "regions is not iterable" in the comp because
of this.

### Webpack ESM strict mode rejects `@one-colored-pixel/canvas`

The canvas package emits ESM dist with extensionless relative imports
("./floodFill" not "./floodFill.js"). Webpack 5 in strict ESM mode rejects
these. The `remotion.config.ts` has the `fullySpecified: false` override
but it only loads under the Remotion CLI, not when `bundle()` is called
programmatically. We pass the same override directly to `bundle()` in
`render.ts`.

### Tondo font fall-through

V2 reels initially used `FONTS.body = "Rooney Sans, ..."` but Rooney isn't
shipped to the worker. The CSS stack fell through to system sans. Fixed by
making `FONTS.body = "Tondo, ..."` (same font for everything, mirrors
AdVideo's precedent). Always include
`<link rel="stylesheet" href={TONDO_FONT_CSS_URL} />` at the top of each
reel comp.

### Vercel CLI env-add adds trailing newlines

`echo "$VAR" | vercel env add` preserves the trailing newline from `echo`.
This silently broke Anthropic + Deepgram + ElevenLabs auth at various
points. Always use `printf "%s"` when piping into `vercel env add`.

### Worker restarts kill in-progress renders

The systemd service restarts when GitHub Action re-deploys. Anything
mid-flight (region-store regen, reel render) gets killed. If you trigger a
long-running operation, do it BEFORE pushing or wait for deploys to settle.

## Operations

### Triggering a fresh image manually

```bash
curl -s -X POST 'https://www.chunkycrayon.com/api/social/demo-reel/produce-v2' \
  -H "Authorization: Bearer $CRON_SECRET" \
  --max-time 200
```

Optional `?variant=text|image|voice` to override rotation. Returns the new
`coloringImageId`. Worker will render in 5-8 min.

### Re-rendering the reel for an existing image

```bash
curl -s -X POST 'http://157.90.168.197:3030/publish/v2' \
  -H "Authorization: Bearer $WORKER_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"coloringImageId":"<id>","variant":"text|image|voice"}'
```

The polling fix means it works even on fresh rows that aren't fully ready
yet. Idempotent: overwrites previous demoReelUrl on the row.

### Repairing a broken regionsJson

If a row has a collapsed palette and you want to retry the labelling pass:

```sql
UPDATE coloring_images
SET "regionMapUrl" = NULL, "regionsJson" = NULL,
    "regionsGeneratedAt" = NULL,
    "regionMapWidth" = NULL, "regionMapHeight" = NULL
WHERE id = '<id>';
```

Then `POST /generate/region-store` with that id. ~3-5 min for the LLM
labelling pass.

### Posting a reel to all platforms manually

```bash
for plat in instagram facebook; do
  curl -s -X POST "https://www.chunkycrayon.com/api/social/post?type=demo-reel&platform=$plat&coloring_image_id=<id>"
  sleep 30
done
```

Pinterest + TikTok require their own steps (Pinterest cron is automatic,
TikTok is manual upload via digest email).

### Previewing captions before posting

```bash
cd apps/chunky-crayon-web
DATABASE_URL='<prod>' pnpm tsx scripts/preview-reel-captions.ts
```

Edit the IDS array at the top of the script to target other rows.

# Blog content strategy

How to grow the Chunky Crayon blog from ~10 posts to 50 without diluting
the ranking signal we already have.

## Why this doc exists

The blog is the only Chunky Crayon surface that's been validated by Google
traffic. PostHog data from May 2026 (last 90 days) shows almost every
non-homepage Google-organic visitor lands on one of four blog posts:

| Blog post slug                         | 90d Google visits |
| -------------------------------------- | ----------------- |
| `adhd-coloring-activities-focus-tools` | 17                |
| `fix-coloring-mistakes-tips`           | 6                 |
| `coloring-activities-long-journeys`    | 4                 |
| `background-coloring-techniques-art`   | 4                 |

Volume is small but the pattern is clear: every winning post answers a
specific real-world problem a parent/teacher has, and uses coloring as
the lens. Generic "fun coloring page tips" posts don't rank.

Blanket-writing 40 more posts would dilute this. This doc picks the
topic clusters and titles that match what's already working, so the next
batch of posts compounds rather than competes.

## What's already there

Generated by `apps/chunky-crayon-worker` daily blog cron, published to
Sanity, rendered at `/[locale]/blog/[slug]`. Topic registry lives at
`packages/coloring-core/src/blog/topics.ts` and prompts at
`packages/coloring-core/src/blog/prompts.ts` — same prompts both apps
use. See [`docs/crons/`](../crons/README.md) for the runtime pipeline.

We have ~10 posts indexed today (build report lists 676+ blog paths
across locales — that's 10ish unique posts × ~6 locales × translation
duplicates).

## Topic-cluster strategy

Five clusters of 8 posts each — 40 posts total. Each cluster targets a
**different parent search context**, not a different keyword variation,
to avoid cannibalising the same SERP.

### Cluster 1 — Coloring for specific needs (high-intent, low competition)

Pattern that's already working (`adhd-coloring-activities-focus-tools`).
Parents Googling for help with a specific kid challenge, coloring is
proposed as one tool. Hits "parent looking for help" intent which is
much more commercial than generic "coloring fun" queries.

1. `coloring-pages-for-sensory-regulation-kids` — for parents of sensory-sensitive kids
2. `coloring-activities-for-anxiety-in-children` — anxiety calm-down activities

## Cross-cluster shape rules

Every post follows the same scaffolding so they compound, not collide:

1. **Hook (50-100 words)** — Real-parent problem, no preamble. The
   parent already Googled it, don't re-explain why coloring is fun.
2. **5-8 actionable tips** — Each one is its own H2. Specific. Photo-able.
3. **Internal links** — At least 3 deep links to existing surfaces:
   - One to `/coloring-pages/<relevant landing slug>` (the new ones we
     just shipped — e.g. an anxiety post links to
     `/coloring-pages/simple-fairy-coloring-pages-for-toddlers` if
     calming/cute is the angle)
   - One to the generator (`/`)
   - One to another blog post in a different cluster (avoid same-cluster
     to keep keyword cannibalisation at zero)
4. **One image or AI-generated header** — bold, on-brand, not stock.
5. **CTA at the end** — "Generate your own coloring page for [topic]"
   linked back to the generator with appropriate prompt prefill if
   possible.
6. **No em dashes in user-facing copy** (per CC voice rules), no "AI"
   mentions in marketing copy.
7. **1200-1800 words.** Below 1000 reads thin; above 2000 dilutes the
   keyword density.

## Implementation cadence

40 posts is a lot. Two ways to ship them, both compatible with the
existing daily blog cron:

### Option A (faster, manual) — 5 posts/week for 8 weeks

Pre-write 5 posts per week and seed them through Sanity directly. Cron
stays running but topic registry favours these 40. Done by ~mid-July 2026.

### Option B (slower, automated) — 1 post/day from this list for 40 days

Add these 40 slugs to `packages/coloring-core/src/blog/topics.ts` at the
top of the priority list. Daily cron picks the next one. Done in ~6 weeks,
no human effort once the topic list is in.

**Recommended:** Option B for first 20 posts, Option A for the remaining
20 where the topic warrants more care (Cluster 1 — needs-specific posts
should not be AI-fired-and-forget because they touch sensitive subjects).

## Measurement

Tag every post in PostHog with `blog_cluster` property (one of:
`needs`, `quiet-time`, `how-to`, `educational`, `celebration`). After
60 days, query Google-organic landing rate by cluster to see which is
compounding fastest, then double down for the next batch.

Don't write batch 3 of 40 (i.e. cross 50 posts) until cluster
performance data is in — flying blind past that point.

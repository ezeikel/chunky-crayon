## 3. App Privacy (Apple Nutrition Label)

> **⚠️ REVISED 2026-06-22 after the build 33 rejection (Guideline 1.3 — Kids
> Category).** Apple rejected build 33 because the iOS app shipped **third-party
> analytics (PostHog: capture + identify + session replay → PostHog Cloud)** AND
> the App Privacy declaration described analytics/tracking-capable data
> collection. For a Kids Category app, third-party analytics SDKs that can
> collect/transmit identifiable or device info are not allowed at all.
>
> **Fix shipped on iOS (Android unchanged — Play app already live with
> PostHog):**
>
> - PostHog is **disabled on iOS** — the client is never constructed, no
>   `PostHogProvider`, no events, no session replay, no device/anonymous id
>   leaves an iOS device (`lib/posthog.ts`, `providers.tsx`,
>   `utils/analytics.ts`).
> - Sentry crash reporting **stays**, but on iOS `Sentry.setUser({ id })` only —
>   **no email or name** sent to Sentry (`contexts/AuthContext.tsx`).
>
> The declaration below reflects the **post-fix iOS build**. The old version
> (with PostHog/Analytics rows) is what got rejected — do NOT reinstate it.
> **Verified against source 2026-06-22.** Click-through key for App Store
> Connect → App Privacy. (Android Play Data Safety is separate — see
> `PLAY_DATA_SAFETY_ANSWERS.md`, which keeps PostHog.)

### Global toggles (answer these first)

tracking = No** · **Not required to use the app** (except where noted). **No
data type lists "Analytics" as a purpose** — there is no analytics SDK on iOS
anymore.

| Diagnostics                | Crash Data            | App Functionality | Sentry native + JS crashes; `sendDefaultPii: false`; `enabled: !__DEV__`.                                                                                                            |

| Financial Info             | Purchase History      | App Functionality | RevenueCat subs (Splash/Rainbow/Sparkle) + credit packs. Apple processes payment; no card data in app. No `collectDeviceIdentifiers`.                                                |
| Contact Info               | Email Address         | App Functionality | **Optional** — Sign in with Apple / Google only. **Not sent to any third-party SDK on iOS** (Sentry gets id only; no PostHog). Used for account management.                          |

| User Content               | Photos or Videos      | App Functionality | **Optional** — parent-gated Image mode only. Photo sent to our API (`/photo-to-coloring`) to generate a coloring page; never sent to any analytics/crash SDK.                        |

Usage Data (Product Interaction, Other Usage Data — **no analytics SDK on

content, Surroundings.

### Third-party partners (processors — not used for tracking)

When App Privacy asks which partners receive each data type, assign (iOS —
**PostHog is NOT a partner here**):

| Partner         | Receives                                                                              | Never used for        |
| --------------- | ------------------------------------------------------------------------------------- | --------------------- |
| Sentry (US)     | Crash Data, Performance Data, Other Diagnostic Data, **User ID only** (no email/name) | Tracking, advertising |
| RevenueCat (US) | Purchase History, User ID                                                             | Tracking, advertising |

**Our backend API** receives User Content (photos, audio, text prompts),
Email/Name (on sign-in), and saved artwork — declare under App Functionality on
those rows; storage is Neon PostgreSQL + Cloudflare R2. It is our own
first-party backend, not a third-party data partner.
